Laborator 8 – Aprofundare PHP: Cazuri reale, gestionare erori, securitate

1. Validarea datelor din formulare

Orice aplicatie web primeste date de la utilizator. Acestea trebuie validate atat pe client (JavaScript, optional) cat si pe server (obligatoriu). Reguli comune: campuri obligatorii, format email, lungime minima, numere in interval.

<?php
$errors = [];
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $nume = trim($_POST['nume'] ?? '');
    $email = trim($_POST['email'] ?? '');
    $varsta = $_POST['varsta'] ?? '';
    
    if (strlen($nume) < 3) {
        $errors[] = 'Numele trebuie sa aiba minim 3 caractere.';
    }
    if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
        $errors[] = 'Adresa de email invalida.';
    }
    if (!is_numeric($varsta) || $varsta < 18 || $varsta > 120) {
        $errors[] = 'Varsta trebuie sa fie intre 18 si 120 ani.';
    }
    
    if (empty($errors)) {
        // procesare date curare
        echo 'Date valide. Continuam.';
    }
}
?>
Functii noi:
- trim() – elimina spatiile albe (sau alte caractere) de la inceputul si sfarsitul unui string.
- filter_var() – filtreaza o variabila cu un filtru specificat; returneaza valoarea filtrata sau false.
- FILTER_VALIDATE_EMAIL – constanta folosita ca filtru pentru a verifica daca un string este o adresa de email valida.
- is_numeric() – verifica daca o variabila este un numar sau un string numeric.

2. Upload de fisier sigur

Incarcarea de imagini sau documente necesita restrictii de tip, dimensiune si stocare in folder protejat.

<?php
$target_dir = "uploads/";
$target_file = $target_dir . basename($_FILES["file"]["name"]);
$imageFileType = strtolower(pathinfo($target_file, PATHINFO_EXTENSION));
$maxSize = 2 * 1024 * 1024; // 2MB
$allowed = ['jpg', 'png', 'jpeg', 'gif'];

if (isset($_POST["submit"])) {
    if (!in_array($imageFileType, $allowed)) {
        echo "Doar imagini JPG, PNG, JPEG, GIF sunt permise.";
    } elseif ($_FILES["file"]["size"] > $maxSize) {
        echo "Fisierul este prea mare (max 2MB).";
    } elseif (move_uploaded_file($_FILES["file"]["tmp_name"], $target_file)) {
        echo "Fisier incarcat cu succes: " . htmlspecialchars(basename($_FILES["file"]["name"]));
    } else {
        echo "Eroare la upload.";
    }
}
?>
Functii noi:
- $_FILES – variabila superglobala care contine informatii despre fisierele incarcate printr-un formular cu metoda POST (nume, tip, dimensiune, locatie temporara etc.).
- basename() – returneaza numele ultimei componente dintr-o cale (de ex. din "uploads/poza.jpg" returneaza "poza.jpg").
- pathinfo() – returneaza informatii despre o cale: director, nume de baza, extensie. Folosim PATHINFO_EXTENSION pentru a obtine extensia.
- move_uploaded_file() – muta un fisier incarcat din locatia temporara ($_FILES["file"]["tmp_name"]) in destinatia finala. Este o masura de securitate.
- htmlspecialchars() – converteste caracterele speciale ( <, >, ", ' ) in entitati HTML, prevenind XSS.
Nu stoca niciodata fisierele intr-un director accesibil direct din browser daca contin date sensibile. Redenumeste fisierul (de ex. cu uniqid()) pentru a evita suprascrierea si atacurile de tip path traversal.

3. Gestionarea sesiunilor pentru utilizatori autentificati

Sesiunile mentin starea utilizatorului dupa login. Exemplu simplu: login cu nume si parola hardcodata, apoi acces la date personale.

// login.php
session_start();
if ($_POST['username'] === 'admin' && password_verify($_POST['password'], '$2y$10$...')) {
    $_SESSION['user_id'] = 1;
    $_SESSION['username'] = 'admin';
    header('Location: dashboard.php');
    exit;
}

// dashboard.php
session_start();
if (!isset($_SESSION['user_id'])) {
    header('Location: login.php');
    exit;
}
echo 'Bine ai venit, ' . htmlspecialchars($_SESSION['username']);
Functii noi:
- session_start() – porneste sau reia o sesiune existenta. Trebuie apelata inainte de orice output HTML.
- $_SESSION – variabila superglobala care stocheaza datele sesiunii sub forma unui array.
- password_verify() – verifica daca o parola in text clar corespunde cu un hash creat de password_hash(). Returneaza true sau false.
- header() – trimite un antet HTTP brut (folosit aici pentru redirectare).
- exit – opreste executia scriptului (recomandat dupa redirect).

Pentru logout: session_destroy(); si eliminare cookie de sesiune.

Functii suplimentare pentru sesiuni:
- session_destroy() – distruge toate datele asociate sesiunii curente (dar nu sterge cookie-ul de sesiune).
- unset($_SESSION['variabila']) – sterge o singura variabila de sesiune.

4. Prevenirea SQL Injection si XSS

Atacurile SQL injection exploateaza concatenarea directa a datelor in query-uri. Solutia: prepared statements (PDO sau MySQLi).

// MySQLi cu prepared statement
$stmt = $conn->prepare("SELECT * FROM users WHERE email = ?");
$stmt->bind_param("s", $email);
$stmt->execute();
$result = $stmt->get_result();

// XSS (Cross-site scripting) – se previne cu htmlspecialchars()
echo htmlspecialchars($user_input, ENT_QUOTES, 'UTF-8');
Functii si metode noi (MySQLi obiect):
- prepare() – pregateste o declaratie SQL pentru executie, returnand un obiect mysqli_stmt.
- bind_param() – leaga variabilele la parametrii locului (?) din interogare. Primul argument specifica tipurile: "s" pentru string, "i" pentru integer, "d" double, "b" blob.
- execute() – executa interogarea pregatita.
- get_result() – obtine un obiect mysqli_result din interogarea executata (pentru SELECT).
- htmlspecialchars() – deja explicata la upload.

5. Tratarea erorilor si debugging

In dezvoltare, afisam erorile; in productie, le logam fara a le afisa utilizatorului.

// in development
ini_set('display_errors', 1);
error_reporting(E_ALL);

// in productie
ini_set('display_errors', 0);
ini_set('log_errors', 1);
error_log("Eroare detalii...", 3, "/path/to/error.log");

// functii utile: var_dump(), print_r() pentru inspectie
Functii noi:
- ini_set() – seteaza valoarea unei optiuni de configurare PHP in timpul executiei.
- error_reporting() – seteaza nivelul de raportare a erorilor (E_ALL raporteaza toate erorile).
- error_log() – trimite un mesaj de eroare catre jurnalul definit (fisier, syslog etc.).
- var_dump() – afiseaza informatii detaliate despre una sau mai multe variabile (tip si valoare).
- print_r() – afiseaza informatie lizibila despre o variabila (util pentru array-uri).

6. Exemplu real: formular de contact cu stocare in fisier CSV

Simulam o cerere de suport client. Se valideaza, se salveaza intr-un fisier CSV si se trimite un email (cu mail() simplu).

<?php
if ($_POST) {
    $nume = htmlspecialchars($_POST['nume']);
    $mesaj = htmlspecialchars($_POST['mesaj']);
    $email = filter_var($_POST['email'], FILTER_SANITIZE_EMAIL);
    
    if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
        die('Email invalid');
    }
    $linie = [$nume, $email, $mesaj, date('Y-m-d H:i:s')];
    $handle = fopen('contacte.csv', 'a');
    fputcsv($handle, $linie);
    fclose($handle);
    
    mail($email, "Confirmare primire", "Am primit mesajul tau. Iti vom raspunde in 24h.");
    echo "Mesaj trimis cu succes.";
}
?>
<form method="post">
    Nume: <input type="text" name="nume" required>
    Email: <input type="email" name="email" required>
    Mesaj: <textarea name="mesaj"></textarea>
    <button type="submit">Trimite</button>
</form>
Functii noi:
- fopen() – deschide un fisier sau o adresa URL. Modul 'a' (append) deschide fisierul pentru scriere, creandu-l daca nu exista, si plaseaza cursorul la sfarsit.
- fputcsv() – scrie o linie sub forma CSV intr-un fisier deschis.
- fclose() – inchide un fisier deschis anterior cu fopen().
- mail() – trimite un email (necesita configuratia SMTP in php.ini).
- die() – opreste executia scriptului si afiseaza un mesaj.

7. Bune practici in PHP

  • Foloseste nume sugestive pentru variabile si functii (ex. $userEmail, nu $a).
  • Separă logica de prezentare (nu amesteca HTML si PHP in exces; foloseste template-uri sau MVC ~Model-View-Controller~).
  • Nu folosi functii deprecated (ex. mysql_*). Foloseste MySQLi sau PDO.
  • Activeaza error reporting in dezvoltare, dezactiveaza-l in productie.
  • Valideaza totdeauna input-urile si escape output-ul (htmlspecialchars).
  • Foloseste hash pentru parole (password_hash, password_verify).
Functii utile pentru parole:
- password_hash($parola, PASSWORD_DEFAULT) – creeaza un hash securizat al parolei.
- password_verify($parola, $hash) – verifica parola fata de hash.
