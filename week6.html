Săptămâna 6 – API-uri, AJAX și jQuery

1. Introducere în API-uri

Un API (Application Programming Interface) este o interfață care permite aplicațiilor să comunice între ele. În context web, API-urile sunt accesate prin HTTP, iar datele sunt adesea returnate în format JSON.

API-urile permit:

  • Preluarea datelor de pe un server distant.
  • Trimiterea de date pentru procesare.
  • Actualizarea sau ștergerea de resurse.
  • Autentificarea și autorizarea.

Structura unui API și REST

REST (Representational State Transfer) este o arhitectură care folosește HTTP în mod standard. Un API RESTful expune resurse (posts, users, comments) accesibile prin URL-uri și operate prin metode HTTP.

Exemplu de structură REST:

  • GET /posts – Obține toate postările.
  • GET /posts/1 – Obține postarea cu ID 1.
  • POST /posts – Crează o nouă postare.
  • PUT /posts/1 – Actualizează postarea cu ID 1 complet.
  • PATCH /posts/1 – Actualizează parțial postarea cu ID 1.
  • DELETE /posts/1 – Șterge postarea cu ID 1.

Format JSON

JSON (JavaScript Object Notation) este formatul standard pentru schimbul de date în API-uri. Este ușor de citit și ușor de parsat în JavaScript.

// Exemplu de răspuns API în JSON
{
    "id": 1,
    "title": "Titlu postare",
    "body": "Conținut postare",
    "userId": 1
}

// Array de obiecte
[
    {
        "id": 1,
        "title": "Postare 1",
        "userId": 1
    },
    {
        "id": 2,
        "title": "Postare 2",
        "userId": 1
    }
]

Un exemplu simplu: https://jsonplaceholder.typicode.com/posts/1 returnează obiectul de mai sus în format JSON.

2. AJAX (Asynchronous JavaScript and XML)

AJAX permite să faci cereri HTTP din JavaScript fără a reîncărca pagina. Deși se numește "XML", azi se folosește mai mult JSON. "Asynchronous" este cuvântul cheie – cererea se execută în fundal, iar pagina rămâne responsivă.

Sincron vs Asincron

Cod sincron se execută linie după linie, iar fiecare operație trebuie să se termine înainte de următoarea. Dacă faci o cerere la server care durează 3 secunde, aplicația se blochează pentru 3 secunde – utilizatorul nu poate interacționa cu pagina.

Cod asincron permite operații de lungă durată să se execute în fundal. Restul codului continuă să se execute, iar când cererea se termină, o callback sau Promise-ul se rezolvă. Utilizatorul rămâne în control.

// SINCRON (rău) - pagina se blochează
const response = earnestFetchSync('https://jsonplaceholder.typicode.com/posts/1'); // Aștept 3 secunde!
console.log('Datele sunt:', response); // Abia după 3 secunde ajung aici
console.log('Utilizatorul poate interacționa'); // Și trebuie să aștepte

// ASINCRON (bun) - pagina rămâne responsivă
console.log('Incepe cererea...');
fetch('https://jsonplaceholder.typicode.com/posts/1'); // Nu aștept, continui imediat
console.log('Cererea s-a inițiat, continui'); // Aceasta se execută imediat
// Când cererea se termină, un callback/Promise se va rezolva în fundal

XMLHttpRequest (XHR) – Metoda tradițională

XMLHttpRequest este API-ul original pentru AJAX, disponibil în browser-e de zeci de ani. Sintaxa este mai verboză.

const xhr = new XMLHttpRequest();
xhr.open('GET', 'https://jsonplaceholder.typicode.com/posts/1');

// Verifică starea cererii pe măsură ce progresează
xhr.onreadystatechange = function() {
    console.log('Stare:', xhr.readyState); // 0=init, 1=opened, 2=headers_received, 3=loading, 4=done
};

// Când cererea se termină cu succes
xhr.onload = function() {
    if (xhr.status === 200) {
        const post = JSON.parse(xhr.responseText);
        console.log('Titlu:', post.title);
    } else {
        console.error('HTTP Error:', xhr.status);
    }
};

// Dacă apare o eroare de rețea
xhr.onerror = function() {
    console.error('Eroare de rețea – nu s-a putut conecta la server');
};

// Trimite cererea
xhr.send();
console.log('Cererea a fost trimisă, dar nu aștept răspunsul');

Fetch API – Metoda modernă cu Promises

Fetch este API-ul modern (introdus unde 2015) și este mult mai curat. Returnează Promise-uri, care sunt obiecte care reprezintă o valoare care va fi disponibilă în viitor (sau o eroare).

fetch('https://jsonplaceholder.typicode.com/posts/1')
    .then(response => {
        // response este obiectul răspunsului HTTP
        console.log('Status:', response.status);
        console.log('OK:', response.ok);
        
        // response.json() returnează un alt Promise care se rezolvă cu datele parsate
        return response.json();
    })
    .then(data => {
        // data este obiectul JSON parsat
        console.log('Titlu:', data.title);
    })
    .catch(error => {
        // Dacă apare o eroare la oricare din .then-uri
        console.error('Eroare:', error.message);
    });

console.log('Cererea a fost inițiată');

Promises – Fundația pentru Async/Await

Un Promise reprezentă rezultatul unei operații asincrone care se va completa în viitor. Poate fi:

  • Pending – Operația este încă în progres.
  • Fulfilled (rezolvat) – Operația s-a terminat cu succes, și avem o valoare.
  • Rejected (respins) – Operația s-a terminat cu o eroare.

Poți crea propriile Promise-uri:

// Promise care se rezolvă după 2 secunde
const myPromise = new Promise((resolve, reject) => {
    setTimeout(() => {
        resolve('Operația s-a terminat cu succes!');
    }, 2000);
});

myPromise
    .then(result => console.log(result)) // Se execută după 2 secunde
    .catch(error => console.error(error));

console.log('Aștept Promise-ul...');

// Promise care se respinge
const failPromise = new Promise((resolve, reject) => {
    reject(new Error('Ceva s-a dereglat!'));
});

failPromise
    .then(result => console.log(result))
    .catch(error => console.error('Am prins eroarea:', error.message));

3. Metode HTTP și operații REST

API-urile folosesc diferite metode HTTP pentru operații diferite. Aceasta este conhecurea conventiei REST:

  • GET – Preluarea datelor (nu modifică server-ul, sigură de executat multiple ori).
  • POST – Trimiterea de noi date (creare resurse, modifică server-ul).
  • PUT – Înlocuirea unei resurse întregi (update complet).
  • PATCH – Actualizarea parțială a unei resurse (schimbă doar câteva câmpuri).
  • DELETE – Ștergerea unei resurse (modifică server-ul).

HTTP Status Codes – Înțelegerea răspunsurilor

Fiecare răspuns HTTP include un status code care indică ce s-a întâmplat:

  • 1xx – Informație (rar întâlnit).
  • 2xx – Succes (200 OK, 201 Created, 204 No Content).
  • 3xx – Redirecționare (301, 302, 304).
  • 4xx – Eroare client (400 Bad Request, 401 Unauthorized, 404 Not Found, 429 Too Many Requests).
  • 5xx – Eroare server (500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable).

Exemple cu Fetch pentru diferite metode

// GET (implicit) - Preluare date
fetch('https://jsonplaceholder.typicode.com/posts/1')
    .then(res => res.json())
    .then(data => console.log(data));

// POST - Creare resursa nouă
fetch('https://jsonplaceholder.typicode.com/posts', {
    method: 'POST',
    headers: { 
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
        title: 'Titlu nou',
        body: 'Conținut',
        userId: 1
    })
})
    .then(res => res.json())
    .then(data => console.log('Creat:', data.id)); // Serverul returnează ID-ul nou

// PUT - Update complet
fetch('https://jsonplaceholder.typicode.com/posts/1', {
    method: 'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        title: 'Titlu modificat',
        body: 'Conținut modificat',
        userId: 1
    })
})
    .then(res => res.json())
    .then(data => console.log('Actualizat:', data));

// PATCH - Update parțial (schimbă doar unele câmpuri)
fetch('https://jsonplaceholder.typicode.com/posts/1', {
    method: 'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        title: 'Doar titlul se modifică' // body rămâne neschimbat
    })
})
    .then(res => res.json())
    .then(data => console.log('Parțial actualizat:', data));

// DELETE - Ștergere
fetch('https://jsonplaceholder.typicode.com/posts/1', {
    method: 'DELETE'
})
    .then(res => {
        if (res.ok) console.log('Șters cu succes');
        return res.json();
    });

4. Async/Await – Programare asincronă ușor de citit

Async/await este sintaxa modernă pentru a lucra cu Promise-uri și operații asincrone. Face codul să arate ca șicodurile sincron, dar rămâne asincron în fundal.

Ce înseamnă async?

Cuvântul cheie async declară o funcție ca fiind asincronă. O funcție async:

  • Returnează mereu un Promise.
  • Permite folosirea await în interiorul ei.
  • Dacă returnezi o valoare normală, va fi returnată ca Promise rezolvat.
// Funcție async simplă
async function greet() {
    return 'Salut!';
}

greet(); // Returnează un Promise

greet().then(mesaj => console.log(mesaj)); // "Salut!"

// Echivalent cu:
function greetTraditional() {
    return Promise.resolve('Salut!');
}

Ce înseamnă await?

Cuvântul cheie await pune în pauză execuția funcției async până când Promise-ul se rezolvă. Returnează valoarea din Promise.

async function loadPosts() {
    console.log('1. Incepe cererea');
    
    // await pune în pauză aici până când Promise-ul fetch se rezolvă
    const response = await fetch('https://jsonplaceholder.typicode.com/posts/1');
    console.log('2. Răspunsul a sosit');
    
    // await pune în pauză din nou până când response.json() se rezolvă
    const data = await response.json();
    console.log('3. JSON-ul a fost parsat');
    
    // Acum avem datele și le putem folosi
    console.log('Titlu:', data.title);
    return data;
}

// Chemarea unei funcții async
loadPosts().then(data => {
    console.log('4. Datele au fost procesate în funcție');
});

console.log('5. Aceasta se executează imediat, fără așteptare');

// Ordinea de execuție:
// 5 -> 1 -> 2 -> 3 -> 4

Try/Catch – Tratarea erorilor în async/await

Try/catch permite capturarea erorilor în cod asincron în stil similar cu cod sincron:

async function loadPostsSecure() {
    try {
        console.log('Încerc să iau datele...');
        
        const response = await fetch('https://jsonplaceholder.typicode.com/posts/1');
        
        // Verifică dacă HTTP status indică succes (2xx)
        if (!response.ok) {
            throw new Error(`HTTP ${response.status}: ${response.statusText}`);
        }
        
        const data = await response.json();
        console.log('Succes:', data.title);
        return data;
        
    } catch (error) {
        // Orice eroare din try-ul de mai sus ajunge aici
        console.error('A aparut o eroare:', error.message);
        
        // Poți reacționa diferit în funcție de tip:
        if (error instanceof TypeError) {
            console.error('Eroare de rețea – nu s-a putut conecta');
        } else if (error.message.includes('HTTP')) {
            console.error('Serverul a returnat o eroare');
        }
        
        // Poți reîncerca, afișa mesaj utilizatorului, etc.
        return null;
    } finally {
        // Se execută mereu, indiferent dacă a fost succes sau eroare
        console.log('Cererea a fost finalizată');
    }
}

loadPostsSecure();

Modele practice cu async/await

// Model 1: Preluare simplă
async function getUser(id) {
    const response = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`);
    return response.json();
}

// Unde chemat:
const user = await getUser(1);
console.log(user.name);

// Model 2: Operație care depinde de alta
async function getUserAndPosts(userId) {
    const user = await getUser(userId);
    
    const postsResponse = await fetch(
        `https://jsonplaceholder.typicode.com/posts?userId=${userId}`
    );
    const posts = await postsResponse.json();
    
    return { user, posts };
}

// Model 3: Cereri paralele cu Promise.all()
async function getMultipleUsers(ids) {
    // Inițiază toate cererile în același timp, nu una după cealaltă
    const promises = ids.map(id => getUser(id));
    const users = await Promise.all(promises);
    return users;
}

// Model 4: Retry logic
async function fetchWithRetry(url, maxRetries = 3) {
    let lastError;
    
    for (let attempt = 1; attempt <= maxRetries; attempt++) {
        try {
            const response = await fetch(url);
            if (response.ok) return response.json();
            throw new Error(`HTTP ${response.status}`);
        } catch (error) {
            lastError = error;
            console.log(`Încercarea ${attempt} eșuată, reîncerc...`);
            await new Promise(resolve => setTimeout(resolve, 1000 * attempt)); // Aștept mai mult cu fiecare încercare
        }
    }
    
    throw lastError;
}

Async/Await vs Promises – Comparație

Amândouă fac același lucru, dar async/await este mai ușor de citit:

// Cu Promises (.then)
function loadPostsPromises() {
    return fetch('https://jsonplaceholder.typicode.com/posts/1')
        .then(response => response.json())
        .then(data => {
            console.log(data.title);
            return data;
        })
        .catch(error => {
            console.error('Eroare:', error);
        });
}

// Cu async/await (mai clar!)
async function loadPostsAsync() {
    try {
        const response = await fetch('https://jsonplaceholder.typicode.com/posts/1');
        const data = await response.json();
        console.log(data.title);
        return data;
    } catch (error) {
        console.error('Eroare:', error);
    }
}

// Amândouă se folosesc la fel:
loadPostsPromises().then(data => console.log('Gata'));
await loadPostsAsync();

5. Introducere în jQuery

jQuery este o bibliotecă JavaScript care simplifică manipularea DOM-ului, handle de evenimente și AJAX. Deși azi framework-urile moderne (React, Vue) sunt preferred, jQuery este încă folosit în multe proiecte legacy și pentru prototipuri rapide.

Ce este jQuery?

jQuery oferă o API mai simplă decât vanilla JavaScript pentru:

  • Selectarea și manipularea elementelor DOM.
  • Handle de evenimente (click, submit, input, etc.).
  • Efecte și animații.
  • AJAX requests (mai simplu decât Fetch API).
  • Compatibilitate între browser-e - jQuery gestionează cazurile speciale.

Instalare și includere

Poți înclude jQuery din CDN (Content Delivery Network) direct în HTML. jQuery trebuie inclus înainte de propriile tale scripturi:

<script src="https://code.jquery.com/jquery-3.6.0.min.js"></script>
<!-- Propriile tale scripturi jQuery pot veni după -->
<script src="script.js"></script>

Selectoare și manipulare DOM în jQuery

jQuery folosește sintaxa $() (sau jQuery()) pentru a selecta elemente și returneaza un obiect jQuery pe care poți aplica metode:

// Selectoare - returnează obiecte jQuery
$('#myId');           // ID - selectează elementul cu id="myId"
$('.myClass');        // Clasă - selectează toate elementele cu class="myClass"
$('div');             // Tag - selectează toate div-urile
$('div.active');      // Tag + Clasă - selectează div-uri cu class="active"
$('[data-id="5"]');   // Atribut - selectează după atribut personalizat
$('li:first');        // Pseudoselector - primul element li
$('li:not(.hidden)'); // Negate selector - li care nu au clasa hidden

// Manipulare text și HTML
$('#myId').text('Nou text');                    // Schimbă textul (plain text)
$('#myId').html('<strong>Bold</strong>');       // Schimbă HTML interior (HTML-ul este parsat)
console.log($('#myId').text());                  // Citește textul
console.log($('#myId').html());                  // Citește HTML-ul

// Manipulare clase
$('#myId').addClass('active');                   // Adaugă clasa
$('#myId').removeClass('active');                // Elimină clasa
$('#myId').toggleClass('active');                // Adaugă/elimină în funcție de prezență

// Manipulare CSS
$('#myId').css('color', 'red');                  // O proprietate
$('#myId').css({                                 // Multiple proprietăți
    'color': 'blue',
    'background-color': 'yellow',
    'font-size': '16px'
});

// Manipulare atribute
$('#myId').attr('data-id', '123');               // Set atribut
console.log($('#myId').attr('data-id'));         // Get atribut
$('#myId').removeAttr('data-id');                // Remove atribut

// Manipulare valorile formularelor
$('#myInput').val('Nova valoare');               // Set valoare
console.log($('#myInput').val());                 // Get valoare

// Vizibilitate
$('#myId').show();                               // Arată (display: block)
$('#myId').hide();                               // Ascunde (display: none)
$('#myId').toggle();                             // Toggle între show și hide

// Proprietăți dimensionale
$('#myId').width();                              // Lățime conținutului
$('#myId').height();                             // Înălțime conținutului
$('#myId').outerWidth();                         // Include padding și border
$('#myId').outerHeight(true);                    // Include margin

// Navigare DOM
$('#myId').parent();                             // Elementul parent
$('#myId').children();                           // Copii direcți
$('#myId').find('.child');                       // Descendenți cu selectorul dat
$('#myId').next();                               // Element sibling următor
$('#myId').prev();                               // Element sibling anterior

AJAX cu jQuery – Mult mai simplu decât Fetch!

jQuery oferă metode speciale pentru AJAX care sunt mai concise decât Fetch API. Toate metodele jQuery AJAX returnează un obiect jqXHR care are metode .done(), .fail(), și .always() pentru handle.

// GET simplu - cea mai concisă formă
$.get('https://jsonplaceholder.typicode.com/posts/1', function(data) {
    console.log('Titlu:', data.title);
});

// POST - trimite date și primește răspuns
$.post('https://jsonplaceholder.typicode.com/posts', {
    title: 'Titlu',
    body: 'Conținut',
    userId: 1
}, function(data) {
    console.log('Creat cu ID:', data.id);
});

// AJAX generic cu mai multe opțiuni și handles
$.ajax({
    url: 'https://jsonplaceholder.typicode.com/posts/1',
    type: 'GET',
    dataType: 'json',              // Expect JSON
    timeout: 5000,                 // Timeout după 5 secunde
    success: function(data) {
        console.log('Succes:', data);
    },
    error: function(xhr, status, error) {
        console.error('Eroare:', status, error);
    },
    complete: function() {
        // Se execută mereu, succes sau nu
        console.log('AJAX terminat');
    }
});

// AJAX cu .done() .fail() .always() (stil mai modern)
$.ajax({
    url: 'https://jsonplaceholder.typicode.com/posts',
    type: 'POST',
    contentType: 'application/json',
    data: JSON.stringify({ title: 'Test', body: 'Test', userId: 1 })
})
    .done(function(data) {
        console.log('Succes:', data);
    })
    .fail(function(jqXHR, status, error) {
        console.error('Eșuat:', error);
    })
    .always(function() {
        console.log('Finalizat');
    });

// Scurtătură pentru JSON specificFetch
$.getJSON('https://jsonplaceholder.typicode.com/posts/1', function(data) {
    console.log(data);
});

Event handling cu jQuery

jQuery face event handling-ul mai simplu și consistent în toate browser-ele:

// Click event
$('#myBtn').click(function() {
    console.log('Buton apăsat');
    $(this).text('Apăsat!'); // $(this) referă elementul clic
});

// Input event - se declanșează pe fiecare schimbare
$('#myInput').on('input', function() {
    console.log('Valoare curentă:', $(this).val());
});

// Submit event
$('form').submit(function(e) {
    e.preventDefault(); // Previne trimiterea normală a formularului
    console.log('Formular trimis');
    // Poți trimite datele cu AJAX în loc de reload
});

// Change event - declanșează când elementul pierde focus (select, input, textarea)
$('#mySelect').change(function() {
    console.log('Selectat:', $(this).val());
});

// Hover - enter și leave
$('#myId').hover(
    function() { console.log('Enter'); },
    function() { console.log('Leave'); }
);

// KeyUp și KeyDown
$('#myInput').keyup(function(e) {
    if (e.keyCode === 13) { // Enter key
        console.log('Enter apăsat');
    }
});

// Event delegation - handle de evenimente pentru elemente adăugate dinamic
$(document).on('click', '.dynamicButton', function() {
    console.log('Dynamic button clicked');
});

6. Integrarea Fetch vs jQuery AJAX – Care să folosesc?

Azi, Fetch este preferred pentru cod modern și nou, dar jQuery AJAX este încă util pentru proiecte existente și rapid prototyping. Iată o comparație:

// jQuery AJAX
$.ajax({
    url: 'https://jsonplaceholder.typicode.com/posts',
    type: 'GET',
    success: function(data) {
        console.log(data);
    },
    error: function() {
        console.error('Eroare');
    }
});

// Fetch modern (echivalent)
fetch('https://jsonplaceholder.typicode.com/posts')
    .then(res => res.json())
    .then(data => console.log(data))
    .catch(error => console.error('Eroare', error));

// Fetch cu async/await (stil modern)
async function getData() {
    try {
        const response = await fetch('https://jsonplaceholder.typicode.com/posts');
        const data = await response.json();
        console.log(data);
    } catch (error) {
        console.error('Eroare', error);
    }
}

Când să folosesc jQuery AJAX?

  • Proiecte legacy care deja folosesc jQuery.
  • Trebuie compatibilitate cu browser-e foarte vechi.
  • Vrei cod mai concis pentru AJAX simplu.
  • Folosești deja jQuery pentru DOM manipulation.

Când să folosesc Fetch?

  • Proiecte noi unde jQuery nu este deja inclus.
  • Modern framework-uri (React, Vue, Angular).
  • Ai nevoie de control mai fin asupra request-urilor.
  • Vrei să evitezi dependența externă (jQuery).

8. Bune practici și tratarea erorilor

La lucrul cu API-uri și operații asincrone, este important să gestionezi erorile corespunzător și să urmezi bune practici:

async function fetchData(url) {
    try {
        // Verifică dacă URL-ul este valid și bine format
        if (!url || typeof url !== 'string') {
            throw new Error('URL invalid');
        }
        
        // Setează un timeout pentru cerere (Fetch nu are timeout implicit)
        const controller = new AbortController();
        const timeoutId = setTimeout(() => controller.abort(), 10000); // 10 sec
        
        const response = await fetch(url, { signal: controller.signal });
        clearTimeout(timeoutId);
        
        // Verifică dacă transport HTTP a fost ok (status 2xx)
        if (!response.ok) {
            // Diferite tipuri de erori au înțeles diferit
            if (response.status === 401) {
                throw new Error('Neautorizat – trebuie să te autentifici');
            } else if (response.status === 404) {
                throw new Error('Resursa nu a fost găsită');
            } else if (response.status === 429) {
                throw new Error('Ai trimis prea multe cereri – aștept puțin');
            } else {
                throw new Error(`HTTP ${response.status}: ${response.statusText}`);
            }
        }
        
        // Parsează JSON-ul
        const data = await response.json();
        
        // Poți valida structura datelor
        if (!data || typeof data !== 'object') {
            throw new Error('Datele returnate nu au formatul așteptat');
        }
        
        return data;
        
    } catch (error) {
        // Tratează diferite tipuri de erori
        if (error.name === 'AbortError') {
            console.error('Timeout – serverul a durat prea mult');
        } else if (error instanceof TypeError) {
            console.error('Eroare de rețea – nu s-a putut conecta');
        } else {
            console.error('Eroare:', error.message);
        }
        
        // Poți loga eroarea pentru debug
        console.error('Stack:', error.stack);
        
        // Afișează mesaj utilizatorului
        alert('Nu s-a putut încărca datele. Verifică conexiunea și reîncearcă.');
        
        // Returnează o valoare implicită
        return null;
    }
}

// 1. Caching - nu face aceeași cerere de două ori
const cache = {};
async function fetchWithCache(url) {
    if (cache[url]) {
        console.log('Din cache');
        return cache[url];
    }
    const data = await fetchData(url);
    cache[url] = data;
    return data;
}

// 2. Rate limiting - evită overloading serverului
let requestCount = 0;
const MAX_REQUESTS_PER_MINUTE = 60;
async function fetchRateLimited(url) {
    if (requestCount >= MAX_REQUESTS_PER_MINUTE) {
        throw new Error('Prea multe cereri, aștept');
    }
    requestCount++;
    return fetchData(url);
}

// 3. Loading indicator - informează utilizatorul că ceva se întâmplă
async function fetchWithLoading(url) {
    const loader = document.getElementById('loader');
    loader.style.display = 'block';
    try {
        return await fetchData(url);
    } finally {
        loader.style.display = 'none';
    }
}

// 4. Validare date înainte de trimitere
function validatePost(post) {
    if (!post.title || typeof post.title !== 'string') {
        throw new Error('Titlu invalid');
    }
    if (!post.body || typeof post.body !== 'string') {
        throw new Error('Body invalid');
    }
    if (!post.userId || typeof post.userId !== 'number') {
        throw new Error('UserId invalid');
    }
    return true;
}

// 5. Mensaje user-friendly - nu arăta detalile API în utilizator
async function userFriendlyFetch(url) {
    try {
        return await fetchData(url);
    } catch (error) {
        // Log complet pentru dev
        console.error('[DEBUG]', error);
        
        // Mesaj simplu pentru utilizator
        if (error.message.includes('Neautorizat')) {
            return 'Te rog autentifică-te';
        } else if (error.message.includes('timeout')) {
            return 'Conexiunea este slow. Reîncearcă mai târziu.';
        } else {
            return 'A apărut o problemă. Reîncearcă.';
        }
    }
}
