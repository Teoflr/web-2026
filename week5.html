Săptămâna 5 – JavaScript avansat

1. Condiționale

Condiționalele permit codului să ia decizii în funcție de valoare. Acestea controlează fluxul programului: ce se execută depinde de condiții evaluate la runtime.

În JavaScript, expresia din cadrul if este evaluată la true sau false. Se recomandă folosirea ===/!== pentru verificări stricte (tip și valoare), ca să evităm surprizele compared cu ==/!=.

if / else / else if

if rulează un bloc atunci când condiția este truthy, else if oferă un fallback, iar else se execută când niciuna din condiții nu este îndeplinită. 

const temperatura = 18;

if (temperatura >= 30) {
    console.log("E caniculă");
} else if (temperatura >= 20) {
    console.log("E plăcut");
} else if (temperatura >= 10) {
    console.log("E răcoros");
} else {
    console.log("E frig");
}

Operator ternar

Operatorul ternar este o formă concisă pentru un if/else simplu. Ideal pentru asigurarea valorii sau returnarea imediată.

const recomandare = temperatura >= 20 ? "Tricou" : "Hanorac";
console.log(recomandare);

Switch

const zi = "miercuri";

switch (zi) {
    case "luni":
        console.log("Începe săptămâna");
        break;
    case "miercuri":
        console.log("Mijloc de săptămână");
        break;
    default:
        console.log("Altă zi");
}

2. Truthy/falsy și conversii

Unele valori în JavaScript sunt evaluate ca false în contexte booleene (de exemplu, condițiile if) și sunt numite falsy. Altele care nu sunt falsy sunt truthy. Înțelegerea acestora te ajută să scrii verificări concise și fără buguri.

  • false, 0, "", null, undefined, NaN sunt falsy.
  • Tot ce este altceva este truthy (inclusiv obiecte goale, array-uri goale, string-uri non-empty).

Conversia explicită cu Boolean() va arăta clar ce se consideră truthy/falsy:

console.log(Boolean(0)); // false
console.log(Boolean("test")); // true
console.log(Boolean({})); // true

Operatorii ||, && și ! se bazează pe truthy/falsy. De exemplu, a || b returnează prima valoare truthy sau ultima valoare falsy. 

    const valoare = "";
if (!valoare) {
    console.log("Valoare invalidă");
}

const nume = prompt("Nume") || "Vizitator";
console.log(nume);

3. Obiecte

Obiectele stochează date structurate cu proprietăți și funcții (metode). Sunt utile pentru a modela entități reale (de exemplu, un produs, un utilizator, o locație).

Accesul la proprietăți se poate face cu notația punct (obj.proprietate) sau cu notația bracket (obj["proprietate"]), utilă când numele proprietății vine dintr-o variabilă.

this dintr-o metodă se referă la obiectul curent (la context). Funcția prezentare folosește this.nume și this.varsta.

const student = {
    nume: "Ana",
    varsta: 24,
    hobbyuri: ["citit", "programare"],
    prezentare() {
        return `Student: ${this.nume}, ${this.varsta} ani`;
    }
};

console.log(student.nume); // din punct
console.log(student["varsta"]); // din bracket
console.log(student.prezentare());

student.adresa = "București"; // adăugare dinamică

for (const cheie of Object.keys(student)) {
    console.log(cheie, student[cheie]);
}

Object.keys(obj) returnează un array cu proprietăți, Object.values(obj) returnează valorile, iar Object.entries(obj) returnează perechi [cheie, valoare].

4. Array-uri avansate

Array-urile sunt colecții ordonate și oferă funcții de nivel înalt care simplifică transformările de date:

  • map: aplică o funcție fiecărui element și returnează un nou array.
  • filter: păstrează elementele care trec o condiție.
  • reduce: agregă valorile într-o singură valoare (sumă, produs, concatenare etc.).
  • find: găsește primul element care îndeplinește condiția.
  • some: verifică dacă cel puțin un element corespunde condiției.
  • every: verifică dacă toate elementele corespund condiției.

Aceasta abordare functională reduce nevoia de bucle explicite (for) și face codul mai expresiv și mai ușor de testat.

const numere = [1, 2, 3, 4, 5];
const pare = numere.filter(n => n % 2 === 0);
const patrate = numere.map(n => n * n);
const suma = numere.reduce((acc, n) => acc + n, 0);

console.log(pare, patrate, suma);

5. Funcții avansate

Funcțiile sunt unități de bază pentru reutilizarea logicii. Folosește parametri default pentru a evita undefined și un număr variabil de parametri cu rest/spread.

  • function aduna(a = 0, b = 0) asigură valori implicite când argumentul nu e trecut.
  • suma(...valori) tratează orice număr de argumente ca un array în interiorul funcției.
  • ...vector extinde un array într-o listă de argumente (spread).
  • Destructuring în parametri extrage direct proprietățile obiectului.
function aduna(a = 0, b = 0) {
    return a + b;
}

function suma(...valori) {
    return valori.reduce((acc, v) => acc + v, 0);
}

const vector = [1, 2, 3];
console.log(Math.max(...vector));

const produs = ({ nume, pret }) => `${nume}: ${pret} RON`;
console.log(produs({ nume: "Caiet", pret: 12 }));

6. Scope, hoisting, closure

Scope defineste unde o variabilă poate fi accesată. var are scope la nivel de funcție; let și const au scope de bloc (curly braces).

Hoisting înseamnă că declarațiile de variabile și funcții sunt aduse la începutul blocului de execuție, însă valorile let/const nu pot fi folosite înaintea inițializării (temporal dead zone).

Closure: o funcție internă „memorează” variabilele din contextul extern chiar și după ce funcția externă și-a terminat execuția.

function counter() {
    let value = 0;
    return function() {
        value += 1;
        return value;
    };
}

const next = counter();
console.log(next()); // 1
console.log(next()); // 2

Acest tip de structură este foarte utilă pentru crearea de funcții cu stare internă (de exemplu, încărcare pagini, debounce, timer).

9. localStorage – Persistență pe refresh

localStorage permite salvarea datelor în browser-ul utilizatorului, pe disc local. Acestea rămân disponibile chiar și după ce pagina este închisă și deschisă din nou. Este ideal pentru preferințe, liste, coșuri de cumpărături, și alte date care nu necesită server.

Concepte de bază

  • localStorage.setItem(key, value) – Salvează o pereche cheie-valoare. Valorile sunt mereu string-uri; pentru obiecte trebuie să folosești JSON.stringify().
  • localStorage.getItem(key) – Recuperează valoarea. Dacă cheia nu există, returnează null.
  • localStorage.removeItem(key) – Șterge o înregistrare.
  • localStorage.clear() – Șterge tot localStorage-ul.

localStorage vs sessionStorage

localStorage persistă pagina, sessionStorage se șterge când tab-ul se închide.

JSON.stringify și JSON.parse

localStorage salvează doar string-uri, nu obiecte. Pentru a salva obiecte sau array-uri, trebuie să le convertesti cu JSON.stringify(); pentru a le obține, folosești JSON.parse().

const student = { nume: "Ana", varsta: 24 };

// Salvare
localStorage.setItem('student', JSON.stringify(student));

// Recuperare
const recuperat = localStorage.getItem('student');
const obj = JSON.parse(recuperat);
console.log(obj.nume); // Ana

Exemplu practic: Salvare și încărcare de produse

Iată cum extinzi mini-proiectul pentru a salva produse în localStorage:

const produse = [];
const STORAGE_KEY = 'produse_list'; // Cheie unica pentru localStorage

// Funcție pentru salvare
function salvareLocalStorage() {
    localStorage.setItem(STORAGE_KEY, JSON.stringify(produse));
}

// Funcție pentru încărcare
function incarcareLocalStorage() {
    const salvate = localStorage.getItem(STORAGE_KEY);
    if (salvate) {
        return JSON.parse(salvate);
    }
    return [];
}

// La inițializare, încarcă din localStorage
const saved = incarcareLocalStorage();
produse.push(...saved);
afiseazaProduse(produse);

// După adăugarea unui produs, salvează
form.addEventListener('submit', function(e) {
    e.preventDefault();
    // ...validare...
    produse.push({ nume, pret, categorie });
    afiseazaProduse(produse);
    salvareLocalStorage(); // Salvează noua listă
    form.reset();
});

// Buton pentru a șterge tot
document.getElementById('clearBtn')?.addEventListener('click', () => {
    localStorage.removeItem(STORAGE_KEY);
    produse.length = 0;
    afiseazaProduse(produse);
    console.log('localStorage șters');
});

Limitări și bune practici

  • Limita de spațiu: Majoritatea browser-elor permit ~5-10MB pe domeniu.
  • Doar string-uri: Mereu convertă obiecte cu JSON.
  • Sincron: localStorage este sincron, nu async; pentru volume mari, folosește indexedDB.
  • Cheii unice: Definește constante pentru chei localStorage (e.g., STORAGE_KEY) pentru a evita typo-uri.
  • Validare la incărcare: După JSON.parse, verifică că datele corespund structurii așteptate, în caz că au fost modificate.
