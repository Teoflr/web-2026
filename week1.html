Săptămâna 1 – Configurarea Mediului de Dezvoltare

1. Instalare XAMPP

XAMPP este un pachet care conține Apache (server web), MySQL (baza de date), PHP și alte utilitare necesare pentru dezvoltarea web locală. Urmează instrucțiunile pentru sistemul operativ pe care lucrezi:

Pentru macOS

  1. Deschide site-ul https://www.apachefriends.org/download.html.
  2. Selectează versiunea pentru macOS și descarcă fișierul .dmg.
  3. Deschide fișierul .dmg și urmează instrucțiunile installer-ului.
  4. După instalare, caută XAMPP în Applications folder și deschide-l.
  5. Deschide XAMPP Control Panel (sau "XAMPP Manager") și pornește modulele Apache și MySQL.
  6. Verifică funcționarea: deschide browserul și navighează la http://localhost. Ar trebui să vezi pagina de start XAMPP cu mesajul "It Works!".
  7. Folderul root pentru proiecte web se află în /Applications/XAMPP/htdocs. Acolo îți vei pune fișierele HTML și CSS.

Pentru Windows

  1. Deschide site-ul https://www.apachefriends.org/download.html.
  2. Selectează versiunea pentru Windows și descarcă fișierul .exe.
  3. Rulează fișierul .exe și urmează instrucțiunile installer-ului (recomandă instalarea în C:\xampp).
  4. Dacă Windows cere permisiunea (UAC), acceptă.
  5. După instalare, deschide XAMPP Control Panel din Start menu.
  6. Pornește modulele Apache și MySQL (butonul "Start" lângă fiecare).
  7. Verifică funcționarea: deschide browserul și navighează la http://localhost. Ar trebui să vezi pagina de start XAMPP.
  8. Folderul root pentru proiecte web se află în C:\xampp\htdocs. Acolo îți vei pune fișierele HTML și CSS.

2. Configurare VS Code

  • Instalează Visual Studio Code de la code.visualstudio.com.
  • Deschide-l și adaugă extensii utile:
    • HTML CSS Support
    • Live Server
    • GitLens
    • GitHub Copilot (dacă ai acces)
  • Deschide folderul de proiect (de exemplu htdocs din XAMPP sau un subfolder propriu).
  • Activează Live Server (clic dreapta pe index.html > Open with Live Server) pentru a vedea modificările automat.

3. Inițializare Git și Copilot

Ce este Git și de ce îl folosim?

Git este un sistem de control al versiunilor. Cu ajutorul lui, poți:

  • Salva diferite versiuni ale codului tău ("snapshots").
  • Vedea ce s-a schimbat între versiuni.
  • Lucra colaborativ cu alții fără să vă suprascrieți munca reciprocă.
  • Reveni la o versiune anterioară dacă ceva se rupe.

Concepte Git de bază

  • Repository (repo) – folderul tău de proiect care conține codul și istoricul versiunilor.
  • Commit – salvarea unei versiuni; este ca o fotografie a codului la acel moment.
  • Branch – o copie a codului în care poți experimenta fără să afectezi versiunea principală (main/master).
  • Remote – o copie a repository-ului pe un server (de exemplu GitHub).

Pași pentru inițializare Git

Deschide terminalul integrat în VS Code (⌃` pe macOS sau Ctrl+` pe Windows, sau View > Terminal) și rulează:

git init

Aceasta creează un folder ascuns .git care va ține evidența versiunilor.

git add .

Aceasta pregătește toate fișierele curente pentru salvare ("staging area").

git commit -m "Prima versiune"

Aceasta salvează versiunea cu mesajul "Prima versiune". Mesajul trebuie să descrie ce s-a schimbat. Mesajele de commit trebuie să fie descriptive și în limba pe care ai ales (de exemplu "Adaugă navbar" sau "Actualizează stiluri CSS").

Comenzi Git utile pentru început

  • git status – arată ce fișiere au fost modificate și care sunt pregătite pentru commit.
  • git log – afișează istoricul commit-urilor (apasă Q pentru a ieși).
  • git diff – arată diferențele între versiunea curentă și versiunea salvată anterior.

Conectare la GitHub (opțional)

Dacă vrei să stochezi codul online și să colaborezi, creează un repository pe GitHub.com. După ce creezi repository-ul, rulează în terminal:

git remote add origin https://github.com/username/proiect.git

Aceasta leagă folderul local de repository-ul online (înlocuiește username cu al tău și proiect cu numele repository-ului). 

git push -u origin main

Aceasta trimite versiunea locală online pe branch-ul main. După prima dată, poți doar folosi git push pentru a trimite modificari.

GitHub Copilot

Copilot este o extensie care te ajută să generezi cod mai rapid. După ce o instalezi în VS Code, pur și simplu începe să tastezi și o vei vedea sugerand cod în timp real. Acceptă sugestia cu Tab sau ignor-o și continuă cu propriul cod. Este o unealtă excelentă pentru a învăța noi sintaxe și modele de codare.

4. Crearea primului fișier HTML

Ce sunt etichetele HTML și de ce le folosim?

HTML (HyperText Markup Language) este limbajul care se folosește pentru a crea pagini web. Etichetele (tags) sunt instrucțiuni care spun browserului cum să afișeze conținutul. Fiecare etichetă are o anumită semnificație:

  • Etichetele se scriu cu caractere < și >: <tagname>.
  • Majoritatea etichetelor au o etichetă de deschidere și una de închidere: <tag>...conținut...</tag>.
  • Etichetele descriu ceea ce conțin: <h1> pentru titlu principal, <p> pentru paragraf, etc.
  • Prin folosirea etichetelor corecte, facem pagina mai ușor de înțeles atât pentru browser, cât și pentru motoarele de căutare (SEO – Search Engine Optimization).

Crearea fișierului

În directorul proiectului creează un fișier nou numit index.html. Completează-l astfel:

<!DOCTYPE html>
<html lang="ro">
<head>
    <meta charset="UTF-8">
    <title>Prima pagină</title>
</head>
<body>
    <h1>Bine aţi venit!</h1>
    <p>Aceasta este prima ta pagină HTML.</p>
</body>
</html>

Explicații pentru etichetele din cod

  • <!DOCTYPE html> – spune browserului că documentul este HTML5 (versiunea modernă); trebuie să fie prima linie din fișier.
  • <html lang="ro"> – eticheta de început a întregului document; atributul lang="ro" spune că pagina e în limba română.
  • <head> – secțiune pentru metadate (informații despre pagină care nu se vad direct pe ecran).
  • <meta charset="UTF-8"> – spune browserului ce set de caractere folosim; UTF-8 permite caractere românești (ă, î, ș, ț, etc.) și caractere din alte limbi.
  • <title> – titlul paginii; apare în tab-ul browserului, nu pe pagina în sine; este important pentru SEO și pentru a ajuta utilizatorii să-și amintească despre pagina ta.
  • <body> – conținutul vizibil al paginii.
  • <h1> – titlu de nivel 1 (cel mai important), titlul principal al paginii; Este permis doar un singur h1 pe pagină.
  • <p> – paragraf; folosit pentru blocuri de text.

Salvare și vizualizare

Salvează fișierul (⌘S pe macOS sau Ctrl+S pe Windows). Apoi vizualizează-l în browser:

  • Cu Live Server (recomandat): clic dreapta pe index.html în explorer-ul de fișiere din VS Code → Open with Live Server. Browserul se deschide automat și vei vedea pagina. Orice modificare pe care o faci va apărea instantaneu în browser.
  • Manual: dublu click pe fisier si il deschizi in browser sau copiază fișierul în folderul htdocs (macOS: /Applications/XAMPP/htdocs, Windows: C:\xampp\htdocs) și navighează la http://localhost/index.html în browserul tău.

Exercițiu: Modifică pagina

Acum că ai o pagină funcțională, încearcă să:

  1. Schimbi textul din <h1> și <p> cu propriul text.
  2. Adaugi un alt paragraf: <p>Text nou</p>.
  3. Schimbi titlul din <title> pentru a vedea cum se schimbă în tab-ul browserului.
  4. Salvează și observă cum se schimbă imediat pagina (dacă folosești Live Server).
